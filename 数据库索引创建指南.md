# 📌 数据库索引创建指南

## ⚠️ 重要提示

**这是提升性能最关键的一步！** 

已完成的代码优化可以减少 50% 的网络请求，但**必须配合数据库索引**才能达到最佳效果。

---

## 🎯 需要创建的索引列表

### 1. families 集合（家庭表）

#### 索引 1: members.openid
```
字段名：members.openid
索引类型：单字段索引
排序：升序
用途：查询用户加入的所有家庭
影响范围：90% 的家庭查询操作
```

### 2. babies 集合（宝宝表）

#### 索引 1: familyId
```
字段名：familyId
索引类型：单字段索引
排序：升序
用途：查询某个家庭的所有宝宝
影响范围：首页加载、宝宝列表页
```

#### 索引 2: familyId + createTime (复合索引)
```
第一字段：familyId (升序)
第二字段：createTime (降序)
用途：按家庭排序宝宝列表
```

### 3. records 集合（记录表）⭐ 最重要

#### 索引 1: babyId
```
字段名：babyId
索引类型：单字段索引
排序：升序
用途：查询宝宝的所有成长记录
影响范围：宝宝详情页每次加载
优先级：⭐⭐⭐⭐⭐ (最高)
```

#### 索引 2: babyId + recordDate (复合索引)
```
第一字段：babyId (升序)
第二字段：recordDate (降序)
用途：获取最新记录、按时间排序记录
```

### 4. inviteCodes 集合（邀请码表）

#### 索引 1: code
```
字段名：code
索引类型：单字段索引
排序：升序
用途：验证邀请码
```

### 5. users 集合（用户表）

#### 索引 1: openid
```
字段名：openid
索引类型：单字段索引
排序：升序
用途：每次登录查询用户信息
```

## 📝 操作步骤

### 方法一：腾讯云开发控制台（推荐）

1. **登录控制台**
   - 访问：https://console.cloud.tencent.com/cloudbase
   - 选择你的环境

2. **进入数据库管理**
   - 左侧菜单：数据库 → 数据集合

3. **为每个集合添加索引**
   - 选择集合（如 families）
   - 点击"索引管理"标签
   - 点击"添加索引"按钮
   - 填写索引配置
   - 点击"确定"

4. **等待索引创建完成**
   - 通常需要 1-3 分钟
   - 状态从"创建中"变为"可用"

### 方法二：使用命令行工具（高级）

```bash
# 示例：为 families 集合添加索引
tcb databaseIndex add families \
  --index-name "members_openid_idx" \
  --keys "members.openid:1"
```

## ⚠️ 注意事项

1. **索引不是越多越好**
   - 每个索引占用存储空间
   - 写入数据时需要更新索引
   - 只为核心查询字段创建索引

2. **复合索引的顺序很重要**
   - 遵循"最左前缀原则"
   - 将选择性高的字段放在前面

3. **创建时机**
   - 建议在低峰期创建（晚上或凌晨）
   - 避免在业务高峰期操作

4. **监控索引使用**
   - 定期查看索引使用率
   - 删除未使用的索引

## 🆘 常见问题

### Q1: 创建索引会影响现有数据吗？
A: 不会。索引创建是后台操作，不影响现有数据和业务。

### Q2: 索引创建失败怎么办？
A: 检查字段名是否正确，集合是否存在。如果仍然失败，联系腾讯云技术支持。

### Q3: 需要重启服务吗？
A: 不需要。索引创建完成后立即生效，无需重启。

### Q4: 索引创建需要多长时间？
A: 
- 小数据量（<1 万条）：1-3 分钟
- 中等数据量（1-10 万条）：5-10 分钟
- 大数据量（>10 万条）：10-30 分钟
