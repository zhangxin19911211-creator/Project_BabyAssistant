# 微信小程序自动化测试脚本

## 📋 功能特点

- ✅ **7 大测试模块**: 覆盖 API、功能、性能、优化等全方位测试
- ✅ **自动评分系统**: 综合性能评分，量化优化效果
- ✅ **详细日志输出**: 分组显示测试结果，问题一目了然
- ✅ **灵活配置**: 支持运行全部测试或单项测试
- ✅ **性能基准**: 提供性能参考值，便于持续优化

---

## 🚀 使用方法

### 方法一：在微信开发者工具中运行（推荐）

1. **打开微信开发者工具**
   - 导入项目：`g:\Project\BabyAssistant_wechat_1`
   - 登录云开发环境

2. **打开调试控制台**
   - 点击右上角"调试器"标签
   - 选择"Console"面板

3. **复制粘贴并运行**
   ```javascript
   // 方式 1: 运行所有测试
   require('./minitest/auto-test.js')
   
   // 方式 2: 运行特定测试
   const test = require('./minitest/auto-test.js')
   test.runSpecificTest('api')      // API 测试
   test.runSpecificTest('baby')     // 宝宝管理测试
   test.runSpecificTest('record')   // 记录管理测试
   test.runSpecificTest('index')    // 索引性能测试
   test.runSpecificTest('parallel') // 并行请求测试
   test.runSpecificTest('debounce') // 防抖节流测试
   test.runSpecificTest('performance') // 性能评分
   ```

### 方法二：使用命令行（需要额外配置）

```bash
# 安装依赖（如果需要）
npm install

# 运行所有测试
npm run test:all

# 运行单项测试
npm run test:api
npm run test:baby
npm run test:record
npm run test:performance
```

## 💡 常见问题

### Q1: 测试失败怎么办？
**A**: 
1. 检查错误信息，定位问题模块
2. 确认已登录云开发环境
3. 检查数据库连接是否正常
4. 查看网络请求是否被拦截

### Q2: 性能评分较低如何优化？
**A**:
1. 检查数据库索引是否创建
2. 确认缓存机制是否生效
3. 减少不必要的网络请求
4. 使用并行请求替代串行

### Q3: 缓存测试不生效？
**A**:
1. 清除浏览器缓存后重试
2. 检查 `wx.setStorageSync` 权限
3. 确认缓存 TTL 设置是否正确

### Q4: 如何在 CI/CD 中使用？
**A**:
1. 使用微信开发者工具的命令行工具
2. 配置自动化测试流程
3. 设置性能阈值告警
4. 集成到部署流水线

## 📝 最佳实践

### 1. 定期运行测试
- 每次代码提交前运行
- 每周至少运行一次全面测试
- 性能优化前后对比测试

### 2. 关注性能指标
- API 响应时间 < 500ms
- 数据库查询 < 200ms
- 缓存命中率 > 80%
- 综合评分 >= 80

### 3. 问题排查流程
```
测试失败 → 查看详细日志 → 定位问题模块 
→ 检查相关代码 → 修复问题 → 重新测试验证
```

### 4. 性能优化循环
```
运行测试 → 查看评分 → 识别瓶颈 
→ 实施优化 → 再次测试 → 对比结果
```
